# Domain List Manager

## Project Goal

Разработать простое веб-приложение для управления списками доменов.

Приложение должно объединять пользовательские домены и внешние списки, удалять дубликаты, хранить историю изменений и публиковать итоговый файл `domains.lst` по HTTP.

Этот файл будет использоваться Ruantiblock как внешний список.

Главный приоритет проекта — простота, надёжность и лёгкость сопровождения.

# Architecture

Использовать классическую архитектуру:

```
HTTP
    ↓
Handlers
    ↓
Services
    ↓
Repositories
    ↓
SQLite
```

## Правила

- Handlers отвечают только за HTTP.
- Services содержат всю бизнес-логику.
- Repositories работают только с базой данных.
- Builder не зависит от HTTP.
- Fetcher отвечает только за скачивание данных.
- Parser отвечает только за разбор данных.
- Scheduler только запускает задачи.

Не создавать сложных абстракций без необходимости.

# Build Algorithm

Builder является центральной частью приложения.

Алгоритм работы:

1. Получить все активные Sources.
2. Скачать обновившиеся списки.
3. Распарсить данные.
4. Добавить пользовательские домены.
5. Нормализовать домены.
6. Удалить дубликаты.
7. Отсортировать список.
8. Построить карту пересечений.
9. Построить diff относительно предыдущей версии.
10. Сохранить snapshot.
11. Атомарно записать domains.lst.
12. Обновить статистику.

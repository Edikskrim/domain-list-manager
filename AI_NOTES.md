# AI Notes

## Project Philosophy

Главная цель проекта — простота.

Не использовать сложные архитектурные паттерны.

Не использовать код "на будущее".

Каждая функция должна выполнять одну задачу.

---

## Technology

- Go 1.25+
- SQLite
- database/sql
- Chi
- html/template
- HTMX
- Alpine.js
- PicoCSS

---

## Architecture

HTTP

↓

Handlers

↓

Services

↓

Repositories

↓

SQLite

---

## Rules

Builder не зависит от HTTP.

Fetcher отвечает только за скачивание.

Parser отвечает только за разбор.

Scheduler только запускает задачи.

Repository работает только с БД.

Handler не содержит бизнес-логики.

---

## Output

Главный результат работы —

domains.lst

Файл должен записываться атомарно через временный файл.

---

## Builder

Builder выполняет:

- скачивание
- парсинг
- объединение
- удаление дубликатов
- сортировку
- поиск пересечений
- построение diff
- сохранение snapshot
- публикацию domains.lst

---

## UI

Использовать серверный рендеринг.

Минимум JavaScript.

HTMX предпочтительнее собственного JS.

---

## Development

Никогда не реализовывать несколько Phase одновременно.

После завершения текущего этапа всегда останавливаться.

Не менять архитектуру без объяснения причины.

Перед каждым новым этапом изучать существующий код.
